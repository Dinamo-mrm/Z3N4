POO en Acción:
Mi Primer Sistema de Gestión

Aprende a dejar las funciones sueltas y construye aplicaciones reales usando el patrón Entidad + Gestora.

MÓDULO 01
La Entidad
Clase Tarea
El molde que define qué ES una tarea y cómo se ve desde afuera.
¿Por qué POO?
Con variables sueltas (let nombre = "Estudiar", let id = 1…) pierdes el contexto: ¿a qué tarea pertenece ese nombre? ¿quién puede cambiarlo? Una Clase Entidad agrupa los datos que van juntos, protege los atributos con #privados y expone solo lo necesario mediante Getters. Es la diferencia entre un cajón revuelto y una carpeta etiquetada.
Código Guía
JavaScript ES6+
// ─── Clase Entidad: el "molde" de una Tarea ─────────────────
class Tarea {

  // Campos PRIVADOS: solo existen dentro de la clase
  #id;
  #nombre;
  #categoria;

  constructor(id, nombre, categoria = 'general') {
    this.#id        = id;
    this.#nombre    = nombre;
    this.#categoria = categoria;
  }

  // ── Getters: puerta de salida de solo lectura ─────────────
  get id()        { return this.#id;        }
  get nombre()    { return this.#nombre;    }
  get categoria() { return this.#categoria; }

  // Setter con validación: evita datos corruptos
  set nombre(nuevoNombre) {
    if (nuevoNombre.trim().length < 3)
      throw new Error('El nombre debe tener al menos 3 caracteres');
    this.#nombre = nuevoNombre.trim();
  }

  // Método de representación → útil para renderizar
  toHTML() {
    return `
      <strong>${this.#nombre}</strong>
      <small>[${this.#categoria}]</small>
    `;
  }
}

// ── Uso ────────────────────────────────────────────────────
const t = new Tarea(1, 'Estudiar POO', 'educación');
console.log(t.nombre);     // ✅ 'Estudiar POO'
console.log(t.#nombre);    // ❌ SyntaxError: campo privado

AtributoTipoAccesoPara qué sirve

Reto de Código 🎯
🏆 Reto 01 — Amplía la Entidad
  • Fácil: Añade un atributo #fechaCreacion que se asigne automáticamente con new Date() en el constructor.
  • Medio: Crea un getter resumen que devuelva un string formateado: "[1] Estudiar POO — educación".
  • Difícil: Añade un método estático Tarea.desdeObjeto(obj) que reciba un objeto plano { id, nombre, categoria } y retorne una nueva instancia de Tarea.
MÓDULO 02
El Cerebro
Clase GestionarApp
La clase administradora que organiza, guarda y opera sobre la colección de entidades.
¿Por qué POO?
Sin una clase gestora, tu código está lleno de arrays globales (let tareas = []) y funciones sueltas que los modifican desde cualquier lugar. Cualquier función puede corromper los datos. GestionarApp encapsula el array, expone solo los métodos necesarios y garantiza que los datos solo se modifiquen de forma controlada.
Metáfora: La clase administradora es como el gerente de un banco. Tú no entras a la bóveda directamente — le pides al gerente (agregar, eliminar, buscar) y él gestiona las cuentas (el array privado) siguiendo las reglas del banco.
Código Guía
JavaScript ES6+
class GestionarApp {

  #tareas = [];           // La bóveda del banco → privada
  #contadorId = 1;        // Auto-incremento de IDs
  #contenedor;            // Referencia al nodo del DOM

  constructor(selectorContenedor) {
    this.#contenedor = document.querySelector(selectorContenedor);
  }

  // ── AGREGAR ────────────────────────────────────────────────
  agregar(nombre, categoria) {
    const nueva = new Tarea(this.#contadorId++, nombre, categoria);
    this.#tareas.push(nueva);
    this.renderizar();       // Siempre re-renderiza
    return nueva;            // Retorna la instancia creada
  }

  // ── ELIMINAR ───────────────────────────────────────────────
  eliminar(id) {
    this.#tareas = this.#tareas.filter(t => t.id !== id);
    this.renderizar();
  }

  // ── BUSCAR ─────────────────────────────────────────────────
  buscarPorNombre(termino) {
    const q = termino.toLowerCase();
    return this.#tareas.filter(t =>
      t.nombre.toLowerCase().includes(q)
    );
  }

  // ── RENDERIZAR ─────────────────────────────────────────────
  renderizar(lista = this.#tareas) {
    if (!lista.length) {
      this.#contenedor.innerHTML = '<p class="empty">Sin tareas</p>';
      return;
    }
    this.#contenedor.innerHTML = lista
      .map(t => `
        <div class="item" data-id="${t.id}">
          ${t.toHTML()}
          <button onclick="app.eliminar(${t.id})">✕</button>
        </div>
      `)
      .join('');
  }

  // Getter público: cuántas tareas hay (sin exponer el array)
  get total() { return this.#tareas.length; }
}
Reto de Código 🎯
🏆 Reto 02 — Amplía el Cerebro
  • Fácil: Añade un método obtenerTodos() que retorne una copia del array ([...this.#tareas]), no el original.
  • Medio: Crea filtrarPorCategoria(cat) que llame a renderizar() con el subconjunto filtrado.
  • Difícil: Añade un método editar(id, nuevoNombre) que encuentre la tarea por id y use el setter para actualizar el nombre, manejando el error con try/catch.
MÓDULO 03
Herencia en Práctica
Clase TareaUrgente
Especializa la entidad base sin repetir código usando extends y super.
¿Por qué POO?
Sin herencia crearías una segunda clase copiando todo el código de Tarea — o peor, añadirías un if (esUrgente) dentro de la misma clase. Con herencia, TareaUrgente reutiliza todo de Tarea y solo añade lo que la diferencia: #prioridad. Sigue el principio DRY (Don't Repeat Yourself).
Código Guía
JavaScript ES6+
class TareaUrgente extends Tarea {

  #prioridad;   // Atributo EXTRA — solo en la subclase

  constructor(id, nombre, categoria, prioridad = 'alta') {
    super(id, nombre, categoria);  // ← llama al constructor padre
    this.#prioridad = prioridad;
  }

  get prioridad() { return this.#prioridad; }

  // Sobreescribimos toHTML para añadir indicador visual
  toHTML() {
    const base = super.toHTML();  // reutiliza el HTML del padre
    return `${base}
      <span class="badge urgente">🔴 ${this.#prioridad.toUpperCase()}</span>
    `;
  }

  // Polimorfismo: mismo método, comportamiento diferente
  get esUrgente() { return true;  }
}

// ── Verificación de tipo ────────────────────────────────────
const tu = new TareaUrgente(5, 'Fix bug crítico', 'dev', 'crítica');

console.log(tu instanceof TareaUrgente);  // true
console.log(tu instanceof Tarea);         // true ← hereda!
console.log(tu.nombre);                   // 'Fix bug crítico'
console.log(tu.prioridad);               // 'crítica'

// GestionarApp acepta ambos tipos sin modificación ✅
const app = new GestionarApp('#lista');
app.agregar('Tarea normal', 'general');
// Para urgentes, puedes extender agregar() o usar una subclase de GestionarApp
Reto de Código 🎯
🏆 Reto 03 — Herencia Avanzada
  • Fácil: Añade un método esMayor(otraTareaUrgente) que compare prioridades ("crítica" > "alta" > "media") y retorne true/false.
  • Medio: Crea una tercera clase TareaRecurrente que herede de Tarea y añada #frecuencia (diaria, semanal, mensual). Sobreescribe toHTML() para mostrar un ícono 🔄.
  • Difícil: Modifica GestionarApp para que renderizar() pinte las TareaUrgente con un borde rojo usando instanceof.
MÓDULO 04
DOM + POO
El Formulario Conectado
Captura eventos del formulario y convierte los datos en instancias de clase.
¿Por qué POO?
Sin POO tendrías un addEventListener suelto que lee el DOM, crea un objeto plano y lo mete en un array global. Con POO, el formulario le habla a la clase gestora: el DOM es solo la vista, y la lógica vive encapsulada en GestionarApp. Si mañana cambias el HTML del formulario, la lógica de negocio no se toca.
Código Guía — Conexión DOM ↔ Clase
JavaScript ES6+
// ── 1. Instancia global de la gestora ──────────────────────
const app = new GestionarApp('#lista-tareas');

// ── 2. Referencia al formulario ────────────────────────────
const form = document.querySelector('#form-tarea');

// ── 3. Escuchar el submit ──────────────────────────────────
form.addEventListener('submit', (e) => {
  e.preventDefault();   // ← evita recargar la página

  // 4. Leer valores del formulario
  const nombre     = form.querySelector('#inp-nombre').value;
  const categoria  = form.querySelector('#sel-categoria').value;
  const esUrgente  = form.querySelector('#chk-urgente').checked;

  // 5. Delegar la lógica a la clase gestora
  if (esUrgente) {
    app.agregarUrgente(nombre, categoria, 'alta');
  } else {
    app.agregar(nombre, categoria);
  }

  form.reset();   // 6. Limpiar el formulario
});

// ── GestionarApp ampliada ──────────────────────────────────
// (añadir dentro de la clase)
agregarUrgente(nombre, categoria, prioridad) {
  const nueva = new TareaUrgente(
    this.#contadorId++,
    nombre, categoria, prioridad
  );
  this.#tareas.push(nueva);
  this.renderizar();
  return nueva;
}
Flujo de Datos
  Usuario llena el formulario
         │
         ▼
  evento 'submit' capturado
         │
         ▼
  Leer valores del DOM      ← solo aquí tocamos el DOM
         │
         ▼
  app.agregar(nombre, cat)  ← lógica SOLO en la clase
         │
         ▼
  new Tarea(id, nombre, cat) ← nueva instancia creada
         │
         ▼
  this.#tareas.push(nueva)
         │
         ▼
  this.renderizar()          ← DOM actualizado desde la clase
Reto de Código 🎯
🏆 Reto 04 — DOM Profesional
  • Fácil: Añade validación en el evento submit: si el nombre está vacío, muestra un mensaje de error en el DOM sin usar alert().
  • Medio: Conecta un <input type="search"> con evento input que en tiempo real llame a app.buscarPorNombre() y re-renderice.
  • Difícil: Implementa edición inline: al hacer doble clic en el nombre de una tarea, conviértelo en un <input>. Al confirmar con Enter, llama a app.editar(id, nuevoNombre).
